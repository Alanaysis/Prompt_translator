# product 术语索引

共 39 个术语

- [[AARRR|AARRR]] — 创业公司指标框架：获取、激活、留存、收入、推荐
- [[AB-Testing|AB Testing]] — 对比两个版本（A 和 B）以确定哪个在特定指标上表现更好的实验
- [[Activation|Activation]] — 新用户首次体验到产品核心价值的时刻
- [[Backlog|Backlog]] — 由产品负责人维护的、按优先级排列的功能、缺陷和任务列表
- [[Bounce-Rate|Bounce Rate]] — 只浏览一个页面就离开网站的访客百分比
- [[Churn|Churn]] — 在给定时间段内停止使用产品的用户百分比
- [[Conversion-Rate|Conversion Rate]] — 完成期望操作（购买、注册等）的用户占总访问者的百分比
- [[DAU|DAU]] — 一天内使用产品的独立用户数量
- [[Freemium|Freemium]] — 基础功能免费、高级功能付费的定价模型
- [[Funnel|Funnel]] — 描述用户从认知到转化各阶段的模型，每个阶段都有用户流失
- [[Growth-Hacking|Growth Hacking]] — 通过在营销渠道和产品开发上快速实验，找到最有效增长方式的策略
- [[Jobs-to-be-Done|Jobs to be Done]] — 关注客户想要完成什么「任务」而非产品功能的框架
- [[KPI|KPI]] — 用于评估战略目标达成情况的可量化指标
- [[Kanban|Kanban]] — 使用列（待办、进行中、已完成）和 WIP 限制的可视化工作流管理方法
- [[MAU|MAU]] — 一个月内使用产品的独立用户数量
- [[MVP|MVP]] — 产品最简单的版本，可以发布以测试核心假设并收集用户反馈
- [[NPS|NPS]] — 基于用户推荐意愿（0-10 分）衡量客户忠诚度的指标
- [[North-Star-Metric|North Star Metric]] — 最能体现产品为客户交付的核心价值的单一指标
- [[OKR|OKR]] — 目标设定框架：目标（O）定义要达成什么，关键结果（KR）衡量如何到达
- [[Onboarding|Onboarding]] — 引导新用户理解并从产品中获得价值的过程
- [[PMF|PMF]] — 产品满足强烈市场需求的程度——当你发现无法跟上需求增长的那一刻
- [[Pain-Point|Pain Point]] — 用户遇到的具体问题或挫败感，代表了产品解决方案的机会
- [[Persona|Persona]] — 基于研究数据创建的、代表关键用户群体的虚构角色
- [[Prioritization|Prioritization]] — 根据价值、工作量和紧迫度决定功能或任务的开发顺序
- [[Product-Led-Growth|Product-Led Growth]] — 由产品本身驱动获客、转化和扩展的增长策略
- [[Prototype|Prototype]] — 产品的交互模拟，用于在开发前测试和验证设计决策
- [[Retention|Retention]] — 在给定时间段内继续使用产品的用户百分比
- [[Retrospective|Retrospective]] — 迭代结束时的会议，团队反思哪些做得好、哪些不好、如何改进
- [[Roadmap|Roadmap]] — 产品愿景和方向的高级可视化概览
- [[Scoping|Scoping]] — 定义项目或发布中包含和不包含内容的边界
- [[Scrum|Scrum]] — 管理复杂工作的敏捷框架，包含角色（PO、SM、开发团队）、事件（迭代、站会、回顾）和工件
- [[Sprint|Sprint]] — 团队完成一组工作项的固定时间段（通常 1-4 周）
- [[Stakeholder|Stakeholder]] — 对项目结果有兴趣或受其影响的人（用户、投资人、团队、管理层）
- [[Standup|Standup]] — 团队成员每天简短分享：昨天做了什么、今天要做什么、有什么阻碍
- [[Usability|Usability]] — 用户完成目标的容易程度和直觉性
- [[User-Story|User Story]] — 从最终用户角度描述功能的简短叙述（作为[用户]，我想要[目标]，以便[获得价值]）
- [[Value-Proposition|Value Proposition]] — 清晰阐述产品为目标用户提供的独特价值，以及用户为什么应该选择它
- [[Viral-Loop|Viral Loop]] — 现有用户自然邀请新用户的机制，产生指数级增长
- [[Wireframe|Wireframe]] — UI 布局的低保真可视化表示，展示结构而非详细设计
