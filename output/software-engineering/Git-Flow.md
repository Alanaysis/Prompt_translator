# Git Flow

**别名**: Git 工作流, 分支策略

## 定义

**中文**: 定义特定分支角色（main、develop、feature、release、hotfix）的分支模型

**English**: A branching model that defines specific branch roles (main, develop, feature, release, hotfix)

> 💡 **Agent 提示**: Git Flow = structured branching model. 'Git 工作流'. main (production), develop (integration), feature/*, release/*, hotfix/*.

## 示例

- "用 Git Flow 管理分支"
  → 按照 Git Flow 模型创建和管理不同用途的分支
