# Throughput

**别名**: 吞吐量, QPS, TPS, RPS

## 定义

**中文**: 系统在单位时间内能处理的请求数量或操作数量

**English**: The number of requests or operations a system can handle per unit of time

> 💡 **Agent 提示**: Throughput = requests per second (QPS/TPS/RPS). High throughput ≠ low latency.

## 示例

- "QPS 能到多少？"
  → 每秒能处理多少次查询请求？

## 关联术语

- latency
- 性能优化
- 并发
