# Technical Debt

**别名**: 技术债, 技术债务, Tech Debt

## 定义

**中文**: 由于当前选择了简单/快速的方案而非更优但耗时更长的方案，而导致未来需要额外返工的隐性成本

**English**: The implied cost of future reworking caused by choosing an easy/quick solution now instead of a better approach that would take longer

> 💡 **Agent 提示**: Technical debt = shortcuts taken now that will cost more to fix later. Like financial debt, it accumulates 'interest'.

## 示例

- "先快速实现，技术债以后再还"
  → 先用简单方案上线，承认以后需要重构，但现在赶进度
