# 自然语言与模型偏好 / Natural Language & Model Preferences

> 机器可读数据：[terms.json](./terms.json)

这个领域比较特殊——它不是传统的行业术语库，而是关于**如何跟 AI 有效沟通**的知识库。

## 核心原则

### 中英文混用是正常的

技术交流中中英文混用不是问题，反而是最高效的方式。关键术语用英文更精确：

| 不推荐 | 推荐 | 原因 |
|--------|------|------|
| 持续集成/持续部署 | CI/CD | 英文缩写更通用 |
| 对象关系映射 | ORM | 术语本身就是英文 |
| 应用程序编程接口 | API | 没人说全称 |
| 可以扩展的 | scalable | 一个词 vs 一句话 |

### 模型语言偏好

不同模型在不同语言上的能力不同：

| 模型 | 推理语言 | 代码语言 | 对话语言 | 备注 |
|------|----------|----------|----------|------|
| Claude | EN | EN | 跟随用户 | 英文推理最准确 |
| GPT-4 | EN | EN | 跟随用户 | 类似 Claude |
| DeepSeek | ZH | EN | ZH | 中文推理很强 |
| Qwen | ZH | EN | ZH | 阿里系，中文优秀 |

**实践建议**:
- 代码和关键术语始终用英文
- 推理过程用模型偏好的语言
- 对话跟随用户语言
- 需求描述中文为主，关键术语附英文原文

## 详细条目

### 中英文混用

**是什么**: 中文技术社区中自然地混合中英文交流。这不叫"装"，这叫高效。

**实践**:
- 核心术语保留英文：API、CI/CD、refactoring、MVP
- 描述性内容用中文："这个 API 的 latency 太高了"
- 不要强行翻译：没人说"应用程序编程接口"，都说 API

---

### 术语一致性

**是什么**: 在项目和团队中，同一个概念始终使用同一个词。

**为什么重要**:
- 代码里叫 `user`，文档里叫 `account`，讨论里叫"客户"——谁看得懂？
- 术语不一致是 bug 的温床

**实践**:
- 维护项目术语表
- 代码、文档、讨论使用相同术语
- 新成员入职先看术语表

---

### Context Window
**中文**: 上下文窗口、上下文长度

**是什么**: 模型一次能处理的最大文本量。超出就"失忆"。

**规格参考**:
- Claude: 200k tokens (~15万字)
- GPT-4: 128k tokens (~10万字)
- DeepSeek: 128k tokens

**场景**:
- "上下文窗口不够了" = 对话太长，早期内容被截断
- "长文本处理" = 利用大上下文窗口处理长文档

---

### Prompt Engineering
**中文**: 提示词工程

**是什么**: 设计有效的提示词让 AI 输出更好。不是"随便问问"，是"精心设计"。

**技巧**:
- 明确指令：告诉模型你要什么
- 给例子：用 few-shot 说明格式
- 设约束：告诉模型不要做什么
- 分步骤：复杂任务拆成小步

---

### Few-shot
**中文**: 少样本

**是什么**: 在提示词中给几个输入-输出的例子，让模型学习你想要的格式。

**变体**:
- Zero-shot: 不给例子，直接问
- One-shot: 给一个例子
- Few-shot: 给几个例子（通常 2-5 个）

---

### Hallucination
**中文**: 幻觉，俗称"AI 在瞎编"

**是什么**: AI 生成听起来很对但实际上是编造的信息。自信的胡说八道。

**应对**:
- 不要盲目相信 AI 的事实性陈述
- 用 RAG/grounding 接入真实数据
- 关键信息要人工验证

---

### Grounding
**中文**: 事实锚定

**是什么**: 把 AI 的输出连接到真实、经过验证的数据源。减少幻觉的关键手段。

**方法**:
- RAG：先检索相关文档，再生成回答
- 工具调用：让 AI 使用有事实依据的工具
- 人工审核：关键输出人工验证

---

### RAG
**中文**: 检索增强生成，Retrieval-Augmented Generation

**是什么**: 先从知识库检索相关文档，再让 AI 基于这些文档生成回答。

**流程**:
1. 用户提问
2. 从知识库检索相关文档
3. 把文档 + 问题一起给 AI
4. AI 基于文档生成回答

**优势**: 减少幻觉，回答有据可查

---

### Token
**中文**: 词元

**是什么**: 模型处理文本的基本单位。英文约 0.75 词/token，中文约 1-2 字/token。

**影响**:
- 成本：按 token 计费
- 上下文：context window 以 token 计
- 速度：token 越多越慢

---

### Temperature
**中文**: 温度

**是什么**: 控制输出随机性的参数。

| 值 | 效果 | 适用场景 |
|----|------|----------|
| 0 | 确定性输出 | 代码生成、数据提取 |
| 0.3-0.7 | 平衡 | 一般对话、翻译 |
| 0.8-1.0 | 创意性 | 创意写作、头脑风暴 |

---

### System Prompt
**中文**: 系统提示词

**是什么**: 在对话开始前给模型的"人设"指令。定义模型的角色、行为规则和约束。

**用途**:
- 设定角色："你是一个资深前端工程师"
- 设定规则："回答要简洁，用中文"
- 设定约束："不要编造信息"
